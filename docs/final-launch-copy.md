## H) Technical Interview Explanation
I scoped Wedly as a small but complete system: Next.js App Router UI + server actions, Supabase Auth and Postgres with RLS as the access boundary, and Sentry for monitoring critical flows like create event, RSVP submit, delete, and CSV export. The main engineering trade-off was to keep features minimal while prioritizing correctness and deployment readiness.
